## 第一章：快速入门
-------------------

### 1.1 什么是 addobj.js？

addobj.js 是 openworld-js 的物体管理系统，核心解决两个问题：
- **百万级物体存储**：用 Float32Array 存 100 万个物体只占 ~64MB
- **动态加载卸载**：DPZ 根据距离自动显示/隐藏

### 1.2 最小示例

```javascript
import k from './src/openworld.js';
globalThis.k = k;

k.initWorld('openworldCanv');

// 地面（DPZ=0，永远显示）
k.addPhy({name:'ground-phy', X:0, Y:-2.5, Z:0, width:250, depth:250, height:6});
k.W.cube({n:'ground', x:0, y:-2.5, z:0, w:250, d:250, h:6, b:'7B8B6F'});

// 主角
k.mainVPlayer = k.addPhy({name:'mainPlayer', X:10, Y:3, Z:10, size:1, mass:50, colliGroup:1});
k.W.cube({n:'mainPlayer', b:'FDF9EE'});

// 添加 1000 个 DPZ=3 的物体
for (let i = 0; i < 1000; i++) {
  k.addTABox({
    DPZ: 3,
    X: (Math.random() - 0.5) * 100,
    Y: 1,
    Z: (Math.random() - 0.5) * 100,
    size: 1,
    background: '888888',
    mass: 0
  });
}
```

就这么简单！物体添加后，DPZ 会自动根据玩家距离显示/隐藏。

## 第二章：addobj.js 完整 API
---------------------------

### 2.1 核心函数一览

| 函数 | 说明 |
|------|------|
| `initBodyTypeArray(max)` | 初始化类型化数组 |
| `addTABox({...})` | 添加物体（返回 index）|
| `activeTABox(index)` | 手动激活物体 |
| `hiddenTABox(index)` | 手动隐藏物体 |
| `dynaNodes_lab()` | DPZ 自动更新（每帧调用）|
| `addPhy({...})` | 添加简易物理体 |
| `addBoxes(arr)` | 批量添加 |
| `addGrid({...})` | 网格批量添加 |

### 2.2 addTABox 参数详解

这是最核心的函数，参数最多，但都有默认值，你可以只填需要的。

#### 必选/常用参数：

```javascript
k.addTABox({
  // === 位置与大小
  DPZ: 3,              // [重要] 显示优先级 0-5，默认3
  X: 0, Y: 0, Z: 0, // 位置
  size: 1,               // 统一设置宽高深（可选）
  width: 1, height: 1, depth: 1, // 单独设置（size优先）

  // === 旋转
  rX: 0, rY: 0, rZ: 0, // 欧拉角（度），自动转四元数
  quat: {x,y,z,w},     // 直接传四元数（可选）

  // === 物理
  mass: 0,                // 质量，0=静止，>0=受重力
  colliGroup: 2,           // 碰撞组：1=玩家/地面，2=石头

  // === 渲染
  background: '#888888',     // 颜色，带#或不带都可以
  texture: null,            // 纹理：img元素或id字符串
  mixValue: 0.7,           // 纹理混合度：0=全纹理，1=全颜色
  smooth: 0,              // 平滑着色 0/1
  shape: 'cube',            // 'cube'或'sphere'
  isPhysical: true,         // 是否有物理碰撞
  isVisualMode: true,         // 是否渲染
  isInvisible: false,        // 不渲染但占位置
  activeFunc: (idx) => {},   // 激活时回调

  // === 高级
  tiling: [1, 1],          // 纹理平铺
  isFictBody: false,           // 物理假体（视觉小一圈）
  unCullFace: 0,              // 不剔除背面
  nS: false,                  // 不受光照影响
  isShadow: 0,                  // 是否有阴影
  textureRatio: 1,               // 错误纹理缩放
  customIdx: -1,                // 自定义索引
});
```

#### 返回值：

返回 `index` 数字，用于后续操作这个物体。

### 2.3 DPZ 参数怎么选？

| DPZ | gridsize | 场景距离 | 用途示例 |
|-------|----------|----------|----------|
| 0 | 10000 | 永远显示 | 地面、主角、天空盒 |
| 1 | 200 | 超远 | 巨型建筑轮廓 |
| 2 | 100 | 中远 | 建筑群简模 |
| 3 | 20 | 近距 | 室内家具、树木 |
| 4 | 5 | 很近 | 书本、小物件 |
| 5 | 1 | 极近 | 细节、文字 |

**重要提示**：
- DPZ=0 的物体 gridsize=10000，几乎永远激活
- 每个 DPZ 检查玩家周围 3×3 个格子
- 同时检查 Y 轴范围 `gridsizeY`

### 2.4 碰撞组

| colliGroup | 碰撞掩码 | 说明 |
|------------|----------|------|
| 1 | stoneGroup\|allGroup | 玩家、地面（与所有碰撞）|
| 2 | allGroup | 静止石头（只与玩家碰撞）|

石头间不互撞，性能更好。

### 2.5 手动激活/隐藏

```javascript
const idx = k.addTABox({...});

// 手动激活（一般不用，DPZ 自动）
k.activeTABox(idx);

// 手动隐藏
k.hiddenTABox(idx);
```

激活时会：
- 如果 isPhysical=true：添加 Cannon.js Body
- 如果 isVisualMode=true：调用 W.cube/W.sphere
- 调用 activeFunc(idx)

隐藏时会：
- 移除物理体
- 删除渲染物体
- 调用 deleteFunc(idx)

### 2.6 批量添加

#### addBoxes(arr)

```javascript
const ids = k.addBoxes([
  { X:1, Y:1, Z:1, DPZ:3 },
  { X:2, Y:1, Z:2, DPZ:3 },
  { X:3, Y:1, Z:3, DPZ:3 },
]);
```

#### addGrid({...})

```javascript
// 创建 20×20 的树阵
k.addGrid({
  startX: -50, startZ: -50,
  countX: 20, countZ: 20,
  spacing: 5,  // 间距
  Y: 0,
  DPZ: 3,
  background: '2a8a4a',
  eachCb: (args, ix, iz) => {
    // 每个物体前修改参数
    args.Y = Math.sin(ix * 0.5) * 2;
  }
});
```

## 第三章：DPZ 深度解析
-------------------

### 3.1 DPZ 是什么？

DPZ = Display Priority Zone（显示优先级区域）

核心思想：**远处的物体不渲染，节省性能。

### 3.2 工作原理（30%原理）

```
玩家位置
    ↓
计算每个 DPZ 层级的格子坐标
    ↓
取玩家周围 3×3 个格子
    ↓
在 spatialGrid 中查这些格子里的物体
    ↓
Y 轴范围内的物体 → 激活
    ↓
不再范围内的物体 → 隐藏
```

### 3.3 空间网格数据结构

```javascript
// spatialGrid 的 key 格式：
"DPZ_gridX_gridZ"

// 例如：
"3_2_5"  → DPZ=3, gridX=2, gridZ=5

// 值是 Set(index)，存该格子里的所有物体
```

### 3.4 gridsize 配置

```javascript
// 默认配置（在 chunkManager.js）：
k.gridsize    = [10000, 200, 100, 20, 5, 1]
k.gridsizeY   = [10000, 200, 100, 20, 5, 1]

// 修改配置（在你的代码）：
k.gridsize[3] = 30;   // DPZ=3 改成 30
k.gridsizeY[3] = 1.35; // Y 轴范围
```

### 3.5 DPZ 激活范围图示

```
      ┌─────────────────┐
      │  ●  ●  ●      │  ● = 当前格子
      │  ● [P] ●      │  [P] = 玩家
      │  ●  ●  ●      │
      └─────────────────┘
         DPZ=3 的格子
         检查 3×3 = 9 个格子
```

### 3.6 停止 DPZ

```javascript
k.stopDynaNodes = true;   // 停止
k.stopDynaNodes = false;  // 开启
```

## 第四章：TypeArray 数据结构
-------------------------

### 4.1 为什么用 TypeArray？

- **内存小**：100万物体，普通对象占 100MB+，TypeArray 只占 64MB
- **遍历快**：连续内存，CPU 缓存友好
- **GC 友好**：不会产生垃圾回收压力

### 4.2 数据布局

```
positionsStatus (Float32Array, 8 × MAX_BODIES)
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ x │ y │ z │ qx│ qy│ qz│ qw│sta│ index 0
├───┼───┼───┼───┼───┼───┼───┼───┤
│ x │ y │ z │ qx│ qy│ qz│ qw│sta│ index 1
├───┼───┼───┼───┼───┼───┼───┼───┤
...
└───┴───┴───┴───┴───┴───┴───┴───┘

physicsProps (Float32Array, 8 × MAX_BODIES)
┌───┬───┬───┬───┬───┬───┬───┬───┐
│mas│ w │ h │ d │DPZ│ ? │ ? │ ? │ index 0
├───┼───┼───┼───┼───┼───┼───┼───┤
...
```

### 4.3 状态码 positionsStatus[7]

| 值 | 含义 |
|----|------|
| -1 | 隐藏 |
| 0 | 静止物体 |
| >0 | 有质量的物体 |

### 4.4 直接访问数据（高级）

```javascript
const idx = 1234;
const p = idx * 8;

// 读位置
const x = k.positionsStatus[p];
const y = k.positionsStatus[p+1];
const z = k.positionsStatus[p+2];

// 读物理属性
const mass = k.physicsProps[p];
const dpz  = k.physicsProps[p+4];

// 修改后要调用：
k.positionsStatus[p] = newX;
// 然后 hidden 再 active 才生效
```

### 4.5 indexToArgs

存完整的参数对象：

```javascript
const args = k.indexToArgs.get(idx);
console.log(args.background);  // 读取
args.deleteFunc = () => {}; // 修改
```

### 5.3 带 activeFunc 的互动物体

```javascript
k.addTABox({
  DPZ: 3,
  X: 10, Y: 1, Z: 10,
  size: 2,
  background: 'ff0000',
  mass: 1,
  activeFunc: (idx) => {
    console.log('物体被激活了！index=' + idx);
    // 可以在这里做动画、音效等
    let t = 0;
    const intv = setInterval(() => {
      t += 0.1;
      const obj = k.W.next['T' + idx];
      if (obj) {
        obj.ry = t * 50;
      } else {
        clearInterval(intv);
      }
    }, 50);
  }
});
```

## 第六章：常见问题
-------------------

### 6.1 物体不显示？

检查清单：
1. DPZ 是否太小？走远点试试
2. Y 坐标是否在 gridsizeY 范围内？
3. isInvisible 是否设成 true？
4. isVisualMode 是否设成 false？
5. background 颜色是否设成了和背景一样？

### 6.2 性能不够？

优化方向：
1. 降低 DPZ 高的物体少渲染
2. 增加 DPZ 层级区分更细
3. 用 instances（见 p004 实例化渲染
4. 合理设置 gridsize，别太大

### 6.3 怎么修改已添加的物体？

```javascript
// 方法1：删除再添加
k.hiddenTABox(idx);
k.addTABox({..., customIdx: idx});

// 方法2：直接修改 W.next（如果已激活）
const obj = k.W.next['T' + idx];
if (obj) {
  obj.x = 100;
  obj.b = 'ff0000';
}
```

### 6.4 物体超过 MAX_BODIES？

```javascript
// 扩容（需要重启场景）
k.initBodyTypeArray(2_000_000);
```

### 6.5 碰撞不正常？

检查：
1. mass=0 是静止物体，不会动
2. colliGroup 是否正确
3. 物体大小是否小于 gridsize 的 2 倍
